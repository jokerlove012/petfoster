# 宠物寄养平台数据库设计

## 数据库概述

本项目采用 MySQL 数据库，数据库名为 `pet_foster`，包含 19 张核心数据表，完整覆盖宠物寄养平台的业务需求。

## 数据库使用方法

### 1. 初始化数据库
```bash
# 连接MySQL
mysql -u root -p

# 执行建表脚本
source c:\Users\asus1\Desktop\essay\PetPro\PetJava\src\main\resources\sql\schema.sql;

# 执行测试数据脚本
source c:\Users\asus1\Desktop\essay\PetPro\PetJava\src\main\resources\sql\test_data_large.sql;
```

### 2. 数据库配置
配置文件位置：`src/main/resources/application.yml`

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/pet_foster?useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai
    username: root
    password: 123456
```

## 特色功能

1. **完整的用户体系**：支持宠物主人、机构员工、管理员三种角色
2. **丰富的机构信息**：包含地理位置、营业时间、容量、特色服务等
3. **全面的宠物管理**：宠物档案、健康记录、疫苗信息等
4. **灵活的订单系统**：预约、支付、入住、退房全流程
5. **完整的评价体系**：多维度评分、图片上传、机构回复
6. **钱包系统**：余额管理、充值、提现、交易记录
7. **投诉处理**：投诉提交、处理流程、结果追踪
8. **收藏功能**：用户收藏喜爱的机构

## 索引设计

为提高查询效率，已在以下字段创建索引：
- user表：phone, role
- pet表：user_id
- institution表：status, rating
- booking表：user_id, institution_id, status
- review表：institution_id, user_id
- wallet表：user_id, status
- wallet_transaction表：wallet_id, user_id, type, created_at

## 逻辑删除

所有核心表都支持逻辑删除，使用 `deleted` 字段标记，便于数据恢复和审计。
