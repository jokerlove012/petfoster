## 🚀 如何运行测试

### 运行所有测试
```bash
cd PetJava
mvn test
```

### 运行特定测试类
```bash
# 运行工具类测试
mvn test -Dtest=PriceCalculatorTest

# 运行实体类测试
mvn test -Dtest=BookingTest

# 运行 DTO 测试
mvn test -Dtest=CreateBookingRequestTest

# 运行公共类测试
mvn test -Dtest=ResultTest,PageResultTest
```

### 运行特定包的测试
```bash
# 运行所有 util 测试
mvn test -Dtest="com.pet.util.*Test"

# 运行所有 entity 测试
mvn test -Dtest="com.pet.entity.*Test"
```

## 📝 测试设计原则

1. **FIRST 原则**
   - ✅ Fast (快速): 所有测试在 3 秒内完成
   - ✅ Independent (独立): 测试之间无依赖
   - ✅ Repeatable (可重复): 可在任何环境运行
   - ✅ Self-validating (自验证): 自动判断成功/失败
   - ✅ Timely (及时): 与代码同步开发

2. **AAA 模式**
   - ✅ Arrange (准备): 清晰的测试数据准备
   - ✅ Act (执行): 单一的测试操作
   - ✅ Assert (断言): 明确的预期结果

3. **测试命名规范**
   - ✅ 使用中文描述测试场景
   - ✅ 方法名清晰表达测试意图
   - ✅ 断言消息使用中文说明

## 💡 最佳实践

1. **测试数据准备**
   - 使用 `@BeforeEach` 准备公共测试数据
   - 测试数据清晰明了
   - 避免硬编码魔法数字

2. **断言使用**
   - 使用有意义的断言消息
   - 优先使用专门的断言方法
   - BigDecimal 使用 compareTo 比较

3. **Mock 使用**
   - 只 Mock 外部依赖
   - 验证关键交互
   - 避免过度 Mock
